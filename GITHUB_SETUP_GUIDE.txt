GitHub Actions 自动构建APK详细设置指南
=======================================

## 🚀 设置步骤

### 第1步：创建GitHub仓库

1. **登录GitHub账户**
   访问 https://github.com/login

2. **创建新仓库**
   - 点击右上角 "+" → "New repository"
   - 仓库名称：mouse-clicker-android
   - 选择 "Public" (私有仓库可能涉及Actions分钟数限制)
   - ✅ Initialize this repository with a README
   - ✅ Add .gitignore: Python
   - ✅ Choose a license: MIT
   - 点击 "Create repository"

### 第2步：上传项目文件

使用GitHub Web界面或使用git命令：

```bash
# 克隆仓库到本地
git clone https://github.com/your-username/mouse-clicker-android.git
cd mouse-clicker-android

# 复制所有文件
cp /path/to/your/files/* .

# 或者使用GitHub Desktop上传
```

### 第3步：验证文件结构

确保仓库包含以下文件和目录：
```
mouse-clicker-android/
├── .github/
│   └── workflows/
│       └── build-android.yml
├── mouse_clicker_android.py
├── buildozer.spec
├── README.md
└── GITHUB_SETUP_GUIDE.txt
```

### 第4步：启用Actions

1. **访问仓库页面**: https://github.com/your-username/mouse-clicker-android
2. **点击 "Actions" 标签页**
3. **点击 "I understand my workflows, go ahead and enable them"** (如果提示)
4. **Actions应该自动启用**

### 第5步：触发首次构建

**方法一：修改文件触发**
```bash
git add .
git commit -m "Initial commit - add android clicker"
git push origin main
```

**方法二：手动触发**
1. 在Actions页面
2. 点击 "Build Android APK" 工作流
3. 点击 "Run workflow"
4. 选择分支：main
5. 点击 "Run workflow"

## 📊 监控构建过程

### 构建状态说明：

#### 🟢 运行中 (Running)
- 显示 "In progress"
- 持续时间：15-45分钟
- 这是正常的，因为Android构建很耗时

#### ✅ 成功 (Success)
- 显示 ✅ green checkmark
- "Artifacts" 部分会显示可下载的APK文件
- 文件大小通常在5-15MB

#### ❌ 失败 (Failed)
- 显示 ❌ red x
- 点击查看日志了解失败原因
- 常见失败原因见下方故障排除

## 🎯 优化构建设置

### 减少构建时间：

1. **缓存依赖** (在workflow中添加):
```yaml
- name: Cache Python dependencies
  uses: actions/cache@v3
  with:
    path: ~/.cache/pip
    key: ${{ runner.os }}-pip-${{ hashFiles('**/requirements.txt') }}
```

2. **使用更快的运行环境**:
```yaml
runs-on: ubuntu-22.04  # 使用更新的系统
```

### 自动化发布：

添加自动发布功能，在成功构建后自动创建Release:

```yaml
- name: Create Release
  if: success()
  uses: actions/create-release@v1
  with:
    tag_name: v${{ github.run_number }}
    release_name: Release ${{ github.run_number }}
    draft: false
    prerelease: false
```

## 🔍 故障排除指南

### 构建失败常见原因：

#### 错误1：缺少文件或权限
```
Error: No such file or directory: 'mouse_clicker_android.py'

解决方案：
- 确保所有必需文件都在仓库根目录
- 检查文件名拼写
```

#### 错误2：依赖安装失败
```
Error: Failed to install dependencies

解决方案：
- 检查buildozer.spec中的requirements配置
- 在网络好时重试构建
```

#### 错误3：内存不足
```
Error: Killed - 内存不足

解决方案：
- 工作流已配置8GB swap，通常足够
- 尝试减少requirements依赖
```

#### 错误4：NDK/SDK下载失败
```
Error: Connection timeout

解决方案：
- 重新运行构建（网络波动）
- 等待几分钟后重试
```

## 💡 最佳实践

### 1. 持续性集成
每次推送到main分支时自动构建:
- 确保代码质量
- 快速反馈
- 自动测试

### 2. 版本号管理
建议每次发布新版本时：
```ini
# buildozer.spec
version = 1.0.${{ github.run_number }}
```

### 3. 分支策略
- **main** - 稳定版本，每次推送触发构建
- **develop** - 开发版本，可设置不同的触发条件
- **feature/* ** - 功能分支，测试新功能

### 4. 通知设置
配置构建成功/失败通知：
- 邮件通知
- Slack集成
- 微信通知 (需要webhook)

## 🎁 进阶功能

### 添加自动测试
```yaml
- name: Run tests
  run: |
    python -m pytest tests/
```

### 多平台构建
```yaml
jobs:
  build-android:
    # Android构建配置
    
  build-ios:
    runs-on: macos-latest
    # iOS构建配置
    
  build-windows:
    runs-on: windows-latest
    # Windows构建配置
```

### 构建矩阵
同时测试多个Python版本：
```yaml
strategy:
  matrix:
    python-version: [3.8, 3.9, '3.10']
```
